# 常州跨境电商三中心 - 企业信息管理系统设计文档

## 1. 项目概述

### 1.1 项目背景
常州跨境电商三中心专注于服务常州本土企业开展跨境电商业务。为提升企业服务效率、实现精细化管理，需建设一套企业信息管理系统，用于收集、整理和分析企业数据。

### 1.2 系统目标
- 统一管理企业基础信息和业务数据
- 支持企业转化漏斗分析，追踪企业从潜在客户到入驻孵化的全流程
- 实现服务跟进记录的规范化管理
- 为决策提供数据分析支持

---

## 2. 核心业务模型：企业转化漏斗

### 2.1 漏斗阶段定义

| 阶段编码 | 阶段名称 | 说明 |
|---------|---------|------|
| `POTENTIAL` | 潜在企业 | 已收录但未接触的企业 |
| `NO_DEMAND` | 无明确需求 | 已接触，暂无跨境意向，需持续培育跟进 |
| `NO_INTENTION` | 没有合作意向 | 明确表示不考虑合作，暂时放弃跟进 |
| `HAS_DEMAND` | 有明确需求 | 已接触，有跨境电商或相关服务需求 |
| `SIGNED` | 已签约 | 已签署服务协议 |
| `SETTLED` | 已入驻 | 已注册/入驻三中心载体 |
| `INCUBATING` | 重点孵化 | 成为标杆企业进行深度孵化 |

### 2.2 漏斗流转规则

```
              ┌→ 无明确需求（待跟进）─┬→ 没有合作意向（流失）
潜在企业 ───┤                       ↓
              └→ 有明确需求 ────────→ 已签约 → 已入驻 → 重点孵化
```

- 潜在企业接触后，根据调研结果分流至"无明确需求"或"有明确需求"
- "无明确需求"企业经持续培育后可转化为"有明确需求"，或转为"没有合作意向"
- "没有合作意向"为流失状态，可定期回访尝试激活

### 2.3 漏斗分析指标
- 各阶段企业数量
- 阶段转化率（如：有明确需求→已签约 转化率）
- 平均停留时长
- 流失率分析

## 4. 漏斗分析功能设计

### 4.1 漏斗统计视图

```sql
-- 漏斗各阶段企业数量统计
CREATE VIEW v_funnel_statistics AS
SELECT 
    funnel_stage,
    COUNT(*) as enterprise_count,
    COUNT(*) * 100.0 / SUM(COUNT(*)) OVER() as percentage
FROM enterprise
GROUP BY funnel_stage
ORDER BY FIELD(funnel_stage, 'POTENTIAL','NO_DEMAND','NO_INTENTION','HAS_DEMAND','SIGNED','SETTLED','INCUBATING');
```

### 4.2 漏斗筛选查询示例

```sql
-- 查询有明确需求的企业
SELECT * FROM enterprise WHERE funnel_stage = 'HAS_DEMAND';

-- 查询无明确需求待跟进的企业
SELECT * FROM enterprise WHERE funnel_stage = 'NO_DEMAND';

-- 查询没有合作意向的企业（流失）
SELECT * FROM enterprise WHERE funnel_stage = 'NO_INTENTION';

-- 查询已签约的企业
SELECT * FROM enterprise WHERE funnel_stage IN ('SIGNED', 'SETTLED', 'INCUBATING');

-- 查询已入驻的企业
SELECT * FROM enterprise WHERE funnel_stage IN ('SETTLED', 'INCUBATING');
```

### 4.3 转化率计算

```sql
-- 各阶段转化率
SELECT 
    from_stage,
    to_stage,
    COUNT(*) as conversion_count
FROM funnel_stage_log
WHERE created_at >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY from_stage, to_stage;
```
