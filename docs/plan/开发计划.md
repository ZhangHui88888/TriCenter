## 待开发功能

### 高优先级
- [ ] **企业合作服务档案**：记录三中心与企业的合作与服务历史，含服务类型（培训/政策/孵化/招商/平台对接等）、状态、负责人、描述，支持关联漏斗阶段变更；在企业详情页新增标签页，后端新增 4 个 API（CRUD）。详见 `docs/design/数据分析系统项目需求文档.md § 2.7`。**进度说明**：前端菜单「合作与服务」、`ServiceRecords.tsx` 与 `serviceRecordApi`（`/enterprises/{id}/services`）已占位；后端 Controller 未实现前不对该页做联调，前后端审核基线中本项列为暂缓。

### 中优先级
- [ ] **代码质量与死代码治理**：用静态检查降低多轮 AI 编辑遗留的未使用/重复代码风险。**后端**：IDE SonarLint + Java 未使用声明检查；**前端**：ESLint / TypeScript（如 `noUnusedLocals` 等）与构建告警对齐。**流程约定**：删除或合并可疑代码前执行「查找所有引用」；按模块分期清理，避免与大功能重构混在同一提交。可选：与 CI 或 SonarQube 服务端打通做门禁（视部署成本决定）。**说明**：本条「按模块」指**前后端代码清理与去死代码的粒度**，**不是**前端企业详情页的 Tab/组件模块化（后者见 `docs/plan/数据分析系统开发计划.md` 与已完成里程碑）；**亦不默认包含**后端超大类/高认知复杂度（如 Sonar S3776）的系统性拆分——若要做需另列任务。
