# 服务商管理页面设计

## 页面定位

服务商管理页承担服务商列表查询、筛选、新增、编辑、删除等核心管理职责。位于左侧导航栏「企业管理」下方。

## 页面结构

### 搜索筛选栏

- **关键词搜索**：按服务商名称模糊搜索
- **服务分类**：下拉选择，数据来源 `system_options` 表 `category=provider_category`
- **合作状态**：下拉选择（合作中 / 已暂停 / 已终止）
- **所属区域**：下拉选择，数据来源 `system_options` 表 `category=district`
- **新增服务商**：右侧按钮，弹出新增表单

### 列表表格

| 列名 | 字段 | 宽度 | 说明 |
|------|------|------|------|
| 服务商名称 | name | 200px | 加粗显示 |
| 服务分类 | categoryName | 120px | 分类名称（从 system_options 映射） |
| 所属区域 | district | 100px | |
| 合作状态 | cooperationStatus | 100px | Tag 展示（绿/橙/红） |
| 主要联系人 | primaryContactName/Phone | 150px | 姓名 + 电话副行 |
| 服务评分 | serviceRating | 90px | 来自小程序端同步 |
| 服务次数 | totalServiceCount | 90px | 来自小程序端同步 |
| 操作 | - | 140px | 编辑 / 删除 |

- 分页：默认每页 10 条，支持切换

### 新增/编辑弹窗

- 服务商名称（必填）
- 服务分类（必填，下拉选择）
- 合作状态（可选）
- 服务商简介（可选，多行文本）

## 后端接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/providers` | 分页查询服务商列表 |
| GET | `/api/providers/{id}` | 获取服务商详情 |
| POST | `/api/providers` | 新增服务商 |
| PUT | `/api/providers/{id}` | 编辑服务商 |
| DELETE | `/api/providers/{id}` | 删除服务商（软删除） |

## 数据表

- `providers` — 服务商主表（含 `capability_requirement_ids` JSON 字段，存储可解决的需求ID数组）
- `provider_contacts` — 服务商联系人
- `provider_service_areas` — 服务商服务领域

## 实现说明

- 页面：`frontend/src/pages/ProviderList.tsx`
- 路由：`/providers`
- 后端：`ProviderController` → `ProviderService` → `ProviderMapper`
- API 封装：`frontend/src/services/api.ts` 中 `providerApi`
