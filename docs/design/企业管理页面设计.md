# 企业管理页面设计

## 页面定位

企业管理页承担企业列表查询、批量操作、导入导出与详情跳转等核心入口职责。

页面标题区与筛选区之间为顶部概览区：左侧两张 BankDash 风格卡片展示**与当前筛选条件一致**的聚合指标；右侧为 **BankDash「My Expense」式柱状图**（各阶段企业数，浅灰柱 + 圆顶 + 当前最高阶段青色高亮），数据与列表筛选一致。

## 顶部概览区

### 布局结构

- 左侧：两张并排的信息卡片（深色渐变 + 浅色描边）
- 右侧：阶段分布柱图（ECharts bar，对标 Figma [BankDash Transactions / My Expense](https://www.figma.com/design/7Iiz8h0uBqp3oU8vIEuRgy/BankDash---Dashboard-UI-Kit---Admin-Template-Dashboard---Admin-Dashboard--Community-?node-id=150-225)）
- 桌面端左右分栏，窄屏下自动折叠为上下结构

### 左侧深色卡片：企业数量统计

| 区域 | 文案 | 数据含义 |
|------|------|----------|
| 顶部 | 企业总数 | 当前筛选条件下匹配企业总数（全量，非当前页） |
| 中部左 | 有明确需求企业数 | 漏斗阶段为 `HAS_DEMAND` 的企业数（在同一筛选条件下） |
| 中部右 | 已签约企业数 | 漏斗阶段为 `SIGNED` 的企业数（在同一筛选条件下） |
| 底部 | 企业数量统计 | 卡片主题说明 |

数据接口：`GET /api/enterprises/overview-stats`（与列表相同筛选参数，不传分页即可）。

### 左侧浅色卡片：出口贸易额统计

| 区域 | 文案 | 数据含义 |
|------|------|----------|
| 顶部 | 出口总贸易额（万元） | 筛选结果内，企业「上年外贸营业额」字段非空记录的金额合计 |
| 中部左 | 线下出口贸易额（万元） | 上述合计中，`has_cross_border` ≠ 1 的企业营业额之和 |
| 中部右 | 线上跨境电商出口贸易额（万元） | 上述合计中，`has_cross_border` = 1 的企业营业额之和 |
| 底部 | 出口贸易额统计 | 卡片主题说明 |

**口径说明**：数据库仅有「上年外贸营业额」单一金额字段，暂无独立的线下/线上拆分字段；线上/线下按「是否开展跨境电商」对企业做二分归属，与财务口径可能不完全一致，后续若有字段可再细化。

### 右侧阶段分布（My Expense 柱图）

- 数据：`GET /api/dashboard/analysis-stats` 的 `funnelStats`（`code` / `name` / `count`），与列表、左侧卡片共用同一套筛选参数（不传分页）。
- 展示：横轴为阶段简称（潜在、无需求…），纵轴无刻度线；柱色默认 `#EDF1F7`，**当前筛选结果中企业数最多的阶段**用 `#16DBCC` 高亮并带轻阴影；标题行右侧数字为该最高阶段的企业数（全为 0 时显示 `0`）。

### 刷新时机

筛选、搜索、高级筛选应用/重置、批量删除、单行删除、批量变更阶段、调研 Excel 导入成功后，前端会并行重新请求 `overview-stats` 与 `analysis-stats`，保证与列表筛选一致。

## 视觉参考

- 参考 BankDash 社区稿：左侧银行卡片 + 右侧「My Expense」圆顶柱图（无 Y 轴刻度、单色高亮）。

## 实现说明

- 列表分页：默认每页 **5** 条；工具栏「导入」左侧下拉可选 **5～30** 条/页，与底部分页联动（底部分页器不再提供条数切换）。
- 页面：`frontend/src/pages/EnterpriseList.tsx`
- 样式：`frontend/src/index.css`
- 后端：`GET /api/enterprises/overview-stats`、`GET /api/dashboard/analysis-stats`；统计逻辑与列表共用 `buildFilterWrapper` / `resolveExternalFilters`（见 `EnterpriseServiceImpl`）
