## 3.3 新增企业

### 功能说明
支持两种创建模式：
1. **空白企业创建**：不传任何参数，系统自动生成默认企业名称（格式：`新建企业_时间戳`）
2. **完整信息创建**：传入企业详细信息

### 请求格式
```
POST /api/enterprises
Header: Authorization: Bearer <token>
Content-Type: application/json

# 空白企业创建
Body: {}

# 完整信息创建
Body: {
  "name": "企业名称",
  "creditCode": "统一社会信用代码",
  "province": "江苏省",
  "city": "常州市",
  "district": "新北区",
  "address": "详细地址",
  "industryId": 1,
  "enterpriseType": "生产型",
  "staffSizeId": 1,
  "domesticRevenueId": 1,
  "crossBorderRevenueId": 1,
  "sourceId": 1,
  "website": "https://example.com",
  "contactName": "联系人姓名",
  "contactPhone": "13800138000",
  "contactPosition": "总经理"
}
```

### 响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "name": "新建企业_1738564800000",
    "stage": "POTENTIAL",
    ...
  }
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | String | 否 | 企业名称，不传则自动生成 |
| creditCode | String | 否 | 统一社会信用代码，最多18位 |
| province | String | 否 | 省份 |
| city | String | 否 | 城市 |
| district | String | 否 | 区域 |
| address | String | 否 | 详细地址 |
| industryId | Integer | 否 | 行业ID |
| enterpriseType | String | 否 | 企业类型 |
| staffSizeId | Integer | 否 | 人员规模ID |
| domesticRevenueId | Integer | 否 | 国内营收ID |
| crossBorderRevenueId | Integer | 否 | 跨境营收ID |
| sourceId | Integer | 否 | 来源ID |
| website | String | 否 | 官网 |
| contactName | String | 否 | 联系人姓名 |
| contactPhone | String | 否 | 联系人电话 |
| contactPosition | String | 否 | 联系人职位 |

### 测试用例

| 场景 | 入参 | 预期结果 | 实际结果 | 状态 |
|------|------|----------|----------|------|
| 空白企业创建 | {} | 创建成功，返回自动生成的企业名称 | 创建成功 | ✅ |
| 完整信息创建 | 完整数据 | 创建成功 | 创建成功 | ✅ |
| 仅企业名称 | {"name":"测试企业"} | 创建成功 | 创建成功 | ✅ |
| 重复企业名称 | 已存在的名称 | 400 错误 | 400 错误 | ✅ |
| 企业名称超长 | name超过200字符 | 400 错误 | 400 错误 | ✅ |
| 特殊字符 | name含<script> | 创建成功（存储原值） | 创建成功 | ✅ |
| 带联系人创建 | name+contactName+contactPhone | 创建成功，同时创建联系人 | 创建成功 | ✅ |
| 仅联系人无电话 | contactName有值，contactPhone为空 | 创建成功，不创建联系人 | 创建成功 | ✅ |
| 无Token | - | 403 Forbidden | 403 Forbidden | ✅ |
