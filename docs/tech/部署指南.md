## 二、环境要求

| 依赖 | 版本要求 | 说明 |
|------|----------|------|
| JDK | 17+ | Spring Boot 3.2 运行环境 |
| MySQL | 8.0+ | 主数据库 |
| Redis | 6.0+ | 会话缓存（使用 database 1） |
| Nginx | 1.18+ | 反向代理 + 静态资源服务 |
| Node.js | 18+ | 仅本地打包前端用，服务器不需要 |
| Maven | 3.8+ | 仅本地打包后端用，服务器不需要 |

### 2.1 检查服务器环境

SSH 登录服务器后，逐个检查：

```bash
# 检查 JDK
java -version
# 期望输出：openjdk version "17.x.x" 或更高

# 检查 MySQL
mysql --version
# 期望输出：mysql Ver 8.0.x

# 检查 MySQL 是否运行中
systemctl status mysql
# 或
systemctl status mysqld

# 检查 Redis
redis-server --version
# 期望输出：Redis server v=6.x.x 或更高

# 检查 Redis 是否运行中
systemctl status redis

# 检查 Nginx
nginx -v
# 期望输出：nginx version: nginx/1.18.x 或更高

# 检查 Nginx 是否运行中
systemctl status nginx
```

### 2.2 安装缺失的依赖（如需要）

```bash
# CentOS / RHEL
sudo yum install -y java-17-openjdk java-17-openjdk-devel
sudo yum install -y mysql-server
sudo yum install -y redis
sudo yum install -y nginx

# Ubuntu / Debian
sudo apt update
sudo apt install -y openjdk-17-jdk
sudo apt install -y mysql-server
sudo apt install -y redis-server
sudo apt install -y nginx
```

安装后启动并设置开机自启：

```bash
sudo systemctl start mysql && sudo systemctl enable mysql
sudo systemctl start redis && sudo systemctl enable redis
sudo systemctl start nginx && sudo systemctl enable nginx
```

## 三、环境变量配置

后端通过 `application-prod.yml` 读取环境变量，支持以下配置：

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `DB_USER` | 否 | root | MySQL 用户名 |
| `DB_PASSWORD` | 是 | （空） | MySQL 密码 |
| `REDIS_PASSWORD` | 否 | （空） | Redis 密码（无密码可不设） |
| `JWT_SECRET` | 否 | 内置默认值 | JWT 签名密钥（生产环境建议自定义） |
| `TRICENTER_API_KEY` | 否 | 内置默认值 | 系统间 API Key |

在服务器上设置环境变量（推荐写入 `/etc/environment` 或 systemd 服务文件）：

```bash
# 方式一：写入 /etc/environment（全局生效，需重新登录）
sudo tee -a /etc/environment > /dev/null <<'EOF'
DB_PASSWORD=你的MySQL密码
REDIS_PASSWORD=你的Redis密码
JWT_SECRET=你的自定义JWT密钥
EOF

# 方式二：直接在 systemd 服务中配置（推荐，更安全）
# deploy.sh 会自动创建 tricenter.service，可手动编辑添加 Environment 行
sudo systemctl edit tricenter
# 添加：
# [Service]
# Environment="DB_PASSWORD=你的MySQL密码"
# Environment="REDIS_PASSWORD=你的Redis密码"
```

## 五、本地打包

在本地开发机（Windows）上执行打包，生成部署产物。

### 5.1 前端打包

```bash
cd frontend
npm install
npm run build
```

打包完成后生成 `frontend/dist/` 目录，包含所有静态文件。

### 5.2 后端打包

```bash
cd backend
mvn clean package -DskipTests
```

打包完成后生成 `backend/target/tricenter-0.0.1-SNAPSHOT.jar`（或类似名称）。

## 六、上传到服务器

### 6.1 方式一：Git 拉取（推荐）

在服务器上直接拉取代码：

```bash
cd /opt
sudo git clone 你的仓库地址 tricenter-source
cd tricenter-source
```

然后在服务器上打包（需要服务器安装 Node.js 和 Maven），或者本地打包后只上传产物。

### 6.2 方式二：SCP 上传打包产物（简单直接）

在本地 Windows 上用 PowerShell 或 Git Bash 执行：

```bash
# 上传前端打包产物
scp -r frontend/dist/* root@47.101.133.8:/opt/tricenter/frontend-dist/

# 上传后端 JAR
scp backend/target/tricenter-*.jar root@47.101.133.8:/opt/tricenter/backend/tricenter.jar

# 上传 Nginx 配置
scp nginx/czcrop.top.conf root@47.101.133.8:/etc/nginx/conf.d/
scp nginx/api.czcrop.top.conf root@47.101.133.8:/etc/nginx/conf.d/

# 上传数据库脚本（首次部署需要）
scp docs/sql/schema.sql root@47.101.133.8:/tmp/
scp docs/sql/init.sql root@47.101.133.8:/tmp/
```

### 6.3 方式三：整个项目上传后用 deploy.sh

```bash
# 上传整个项目目录
scp -r . root@47.101.133.8:/opt/tricenter-source/

# SSH 登录服务器后执行部署脚本
ssh root@47.101.133.8
cd /opt/tricenter-source
bash deploy.sh
```

## 七、SSL 证书配置

### 7.1 安装 Certbot

```bash
# CentOS / RHEL
sudo yum install -y certbot python3-certbot-nginx

# Ubuntu / Debian
sudo apt install -y certbot python3-certbot-nginx
```

### 7.2 申请证书

```bash
# 方式一：通配符证书（推荐，覆盖所有子域名）
# 需要 DNS 验证，按提示在阿里云 DNS 添加 TXT 记录
sudo certbot certonly --manual --preferred-challenges dns \
  -d "czcrop.top" -d "*.czcrop.top"

# 方式二：单独申请（自动验证，更简单）
sudo certbot --nginx -d czcrop.top -d www.czcrop.top -d api.czcrop.top
```

证书文件会保存在 `/etc/letsencrypt/live/czcrop.top/` 目录下。

### 7.3 自动续期

```bash
# 测试续期是否正常
sudo certbot renew --dry-run

# Certbot 安装后会自动配置定时任务，无需手动设置
```

## 八、部署执行

### 8.1 使用部署脚本（推荐）

```bash
# SSH 登录服务器
ssh root@47.101.133.8

# 进入项目目录
cd /opt/tricenter-source

# 执行部署
bash deploy.sh
```

`deploy.sh` 会自动完成：
1. 创建部署目录 `/opt/tricenter/`
2. 复制前端静态文件到 `/opt/tricenter/frontend-dist/`
3. 复制后端 JAR 到 `/opt/tricenter/backend/tricenter.jar`
4. 部署 Nginx 配置并重载
5. 创建 systemd 服务 `tricenter.service`
6. 启动后端服务

### 8.2 手动部署

如果不用脚本，手动执行：

```bash
# 创建目录
sudo mkdir -p /opt/tricenter/frontend-dist
sudo mkdir -p /opt/tricenter/backend

# 复制文件（假设产物已上传）
sudo cp -r frontend/dist/* /opt/tricenter/frontend-dist/
sudo cp backend/target/tricenter-*.jar /opt/tricenter/backend/tricenter.jar

# 部署 Nginx 配置
sudo cp nginx/czcrop.top.conf /etc/nginx/conf.d/
sudo cp nginx/api.czcrop.top.conf /etc/nginx/conf.d/
sudo nginx -t && sudo systemctl reload nginx

# 启动后端（前台测试）
cd /opt/tricenter/backend
java -jar -Dspring.profiles.active=prod -Xms256m -Xmx512m tricenter.jar

# 确认正常后 Ctrl+C 停止，改用 systemd 管理（见 deploy.sh 中的服务配置）
```

## 九、部署验证

```bash
# 检查后端服务状态
sudo systemctl status tricenter

# 检查后端日志
sudo journalctl -u tricenter -f --no-pager -n 50

# 检查 Nginx 配置是否正确
sudo nginx -t

# 测试后端 API 是否响应
curl -I https://api.czcrop.top/api/auth/login

# 测试前端是否可访问
curl -I https://czcrop.top
```

部署后的正常日志应以 `INFO/WARN/ERROR` 为主，不再持续打印 MyBatis SQL 明细、`SqlSession` 创建日志和 Spring Security DEBUG 过滤链日志。

若 Redis 未启动或短暂不可用，Dashboard 模块会自动降级为直查数据库：
- 不会因缓存故障导致 `/api/dashboard/*` 返回 500
- 后端仅记录一次降级告警，Redis 恢复后记录一次恢复日志
- 建议仍尽快恢复 Redis，以保证首页统计接口响应速度

浏览器访问验证：
- 前端：`https://czcrop.top`
- 后端 API 文档：`https://api.czcrop.top/doc.html`

## 十一、常用运维命令

```bash
# 重启后端
sudo systemctl restart tricenter

# 停止后端
sudo systemctl stop tricenter

# 查看后端日志
sudo journalctl -u tricenter -f

# 查看最近 Redis / Dashboard 关键日志
sudo journalctl -u tricenter --since "10 minutes ago" | grep -E "Redis|Dashboard|ERROR|WARN"

# 重载 Nginx（修改配置后）
sudo nginx -t && sudo systemctl reload nginx

# 更新前端（重新上传 dist 后）
sudo cp -r /opt/tricenter-source/frontend/dist/* /opt/tricenter/frontend-dist/

# 更新后端（重新上传 JAR 后）
sudo cp /opt/tricenter-source/backend/target/tricenter-*.jar /opt/tricenter/backend/tricenter.jar
sudo systemctl restart tricenter
```

## 十二、回滚方案

1. 后端回滚：将上一版本的 JAR 文件复制回 `/opt/tricenter/backend/tricenter.jar`，重启服务
2. 前端回滚：将上一版本的 dist 文件复制回 `/opt/tricenter/frontend-dist/`
3. 数据库回滚：使用备份的 SQL 文件恢复

建议每次部署前备份：

```bash
# 备份当前版本
sudo cp /opt/tricenter/backend/tricenter.jar /opt/tricenter/backend/tricenter.jar.bak
sudo cp -r /opt/tricenter/frontend-dist /opt/tricenter/frontend-dist.bak
```
