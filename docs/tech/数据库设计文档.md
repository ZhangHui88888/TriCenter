# 数据库设计文档

> 常州跨境电商三中心 - 企业信息管理系统

---

## 数据库选型

- **推荐**: MySQL 8.0+ 或 PostgreSQL 14+
- **字符集**: UTF-8 (utf8mb4)
- **排序规则**: utf8mb4_unicode_ci

---

## SQL 脚本

| 文件 | 说明 |
|------|------|
| `docs/sql/tricenter_schema.sql` | 建表语句（DDL，含 `market_reports` 等全部表） |
| `docs/sql/tricenter_init.sql` | 初始化数据（系统选项、行业分类、产品品类、需求、用户等；**不含** `market_reports` 业务数据） |
| `docs/sql/scripts/` | 上线后迁移脚本（若有） |

> **注**: 销售目标国（country）数据由前端硬编码管理，详见 `frontend/src/data/countries.ts`

## 表结构详情

### 表1: users (用户表)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK AI | 主键 |
| username | VARCHAR(50) NOT NULL UNIQUE | 用户名 |
| password | VARCHAR(255) NOT NULL | 密码(BCrypt加密) |
| name | VARCHAR(50) NOT NULL | 姓名 |
| role | ENUM('admin','manager','user') | 角色，默认 user |
| phone | VARCHAR(20) | 手机号 |
| email | VARCHAR(100) | 邮箱 |
| status | TINYINT | 状态: 1启用 0禁用，默认1 |
| last_login_at | DATETIME | 最后登录时间 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 表2: industry_categories (行业分类表)

三级树形结构，通过 parent_id 和 path 实现层级关系。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK AI | 主键 |
| parent_id | INT | 父级ID，0为顶级 |
| name | VARCHAR(50) NOT NULL | 分类名称 |
| level | TINYINT NOT NULL | 层级：1/2/3 |
| path | VARCHAR(100) | 完整路径，如：1/5/12 |
| sort_order | INT | 排序，默认0 |
| is_enabled | TINYINT | 是否启用，默认1 |
| created_at | DATETIME | 创建时间 |

### 表3: product_categories (产品品类分类表)

结构同 industry_categories，三级树形。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK AI | 主键 |
| parent_id | INT | 父级ID，0为顶级 |
| name | VARCHAR(50) NOT NULL | 品类名称 |
| level | TINYINT NOT NULL | 层级：1/2/3 |
| path | VARCHAR(100) | 完整路径，如：1/5/12 |
| sort_order | INT | 排序，默认0 |
| is_enabled | TINYINT | 是否启用，默认1 |
| created_at | DATETIME | 创建时间 |

### 表4: system_options (系统选项配置表)

系统所有下拉选项、标签的数据源，通过 category 区分类型。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK AI | 主键 |
| category | VARCHAR(50) NOT NULL | 分类: stage/district/enterprise_type/staff_size/revenue/source/region/certification 等 |
| value | VARCHAR(50) NOT NULL | 选项值（唯一键：category+value） |
| label | VARCHAR(100) NOT NULL | 显示名称 |
| color | VARCHAR(20) | 颜色(用于标签显示) |
| sort_order | INT | 排序，默认0 |
| is_enabled | TINYINT | 是否启用，默认1 |
| created_at | DATETIME | 创建时间 |

### 表19: market_reports (市场调研报告)

与后端实体 `com.tricenter.entity.MarketReport` 对应。**一企业仅一行**（`enterprise_id` UNIQUE），基础版与深度版分别存 JSON。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK AI | 主键 |
| enterprise_id | INT NOT NULL UNIQUE FK→enterprises | 企业ID |
| basic_report_data | JSON | 基础版报告 AI 字段 JSON |
| deep_report_data | JSON | 深度版报告 AI 字段 JSON |
| basic_generated_at | DATETIME | 基础版最后生成时间 |
| deep_generated_at | DATETIME | 深度版最后生成时间 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

**初始化**：`tricenter_init.sql` 中**无**本表 INSERT；数据由接口 `PUT /api/market-reports/{enterpriseId}/{version}`（`version` 为 `basic` 或 `deep`）等业务写入。
