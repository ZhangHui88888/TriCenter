# 前后端审核基线

> 创建日期：2026-03-22
> 用途：沉淀 TriCenter 当前前后端契约基线，后续改动任一功能时，只需按命中的模块做局部复审，不必每次全量重查。

---

## 1. 使用方式

1. 先定位本次改动命中的模块与文件。
2. 在“当前模块基线”中找到对应模块，只复查该模块的前端页面、`frontend/src/services/api.ts`、后端 `Controller/DTO`。
3. 按“通用复审清单”逐项核对。
4. 若契约或状态发生变化，直接更新本文对应模块行和“问题清单”。

---

## 2. 通用复审清单

- 页面上的新增、编辑、删除、导入、导出、切换状态按钮，是否都真正调用了接口，而不是只弹提示。
- `frontend/src/services/api.ts` 中的请求方法、路径、参数位置，是否与后端 `Controller` 完全一致。
- 前端表单字段名，是否与后端 Request DTO 字段名一致。
- 前端列表/详情渲染字段名，是否与后端 Response DTO 字段名一致。
- 页面是否误用了相近但语义不同的接口。
- 成功提示是否发生在接口真正成功之后。
- 接口或字段有变更时，是否同步更新 `docs/tech/API设计文档.md`。

## 5. 当前问题清单（2026-03-22）

- **状态**：暂缓（已纳入 `docs/plan/开发计划.md`「企业合作服务档案」）
- 影响范围：`frontend/src/pages/ServiceRecords.tsx`、`frontend/src/services/api.ts`
- 说明：后端未实现前不作为阻塞缺陷；实现后需复审并关闭本条。

### FE-BE-002 跟进统计接口接错 — **已修复（2026-03-22）**

- 跟进记录页改为 `GET /api/follow-ups/stats`，与 `FollowUpStatsResponse` 字段对齐。

### FE-BE-003 跟进新增请求字段不一致 — **已修复（2026-03-22）**

- 创建请求使用 `followType`、`followDate`、`status` 等与后端一致。

### FE-BE-004 跟进列表响应字段不一致 — **已修复（2026-03-22）**

- 列表映射 `followerName`、`followDate`、`followType`、`stageFrom`、`stageTo` 等。

### FE-BE-005 企业详情中的跟进记录未接后端 — **已修复（2026-03-22）**

- 详情页跟进列表与增删改对接 `followUpApi`。

### FE-BE-006 企业详情中的产品/专利按钮未接后端 — **已修复（2026-03-22）**

- 产品/专利增删改对接 `productApi`/`patentApi`；产品品类级联对接 `options/product-categories`。

### FE-BE-007 数据字典页仅操作本地状态 — **部分修复（2026-03-22）**

- 平面字典分类已对接 `options` + `dictionary`；多级树仍为本地数据，见模块基线。

### FE-BE-008 漏斗分析独立模块 — **已下线（2026-03-23）**

- 已移除前端 `FunnelAnalysis` 页与后端 `FunnelController` / `/api/funnel/*`；漏斗**阶段数量**仍在看板与数据分析页展示（`dashboard/funnel`、`analysis-stats` 等）。

## 7. 维护规则

- 每次完成一次局部前后端复审后，只更新命中的模块行，不需要重写整篇文档。
- 若某问题已修复，保留问题编号，状态改为“已修复”，并补充修复日期。
- 若新增模块或页面，必须先在本文增加模块行，再开始实现。
- 若接口契约发生变化，必须同步更新 `docs/tech/API设计文档.md`。
