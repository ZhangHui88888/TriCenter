# 持续学习 Skill

## 触发条件
- 用户明确要求深度总结或复盘
- 出现了反复出错的模式（同一问题出现 2 次以上）
- 做出了重要架构决策需要记录

## 执行步骤

1. **回顾本次会话**：识别：
   - 犯过的错误（尤其是反复出现的）
   - 发现的有效模式和快捷方式
   - 做出的架构决策及原因

2. **追加到 `.cursor/memory/learnings.md`**（不覆盖历史）：

   格式：
   ```
   ## YYYY-MM-DD — [主题简述]

   ### 问题
   [什么出了错]

   ### 根因
   [为什么会出现]

   ### 解决方案
   [怎么修复 / 最佳实践]

   ### 防范建议
   [是否需要新增 Rule 自动防范]
   ```

3. **评估是否需要新 Rule**：如果某个错误出现 2 次以上，主动建议创建 `.cursor/rules/` 规则自动防范，并给出规则草稿。

4. **压缩 context.md**：将本次进度更新为最小可恢复摘要。

## 与 memory.mdc 的关系
- `memory.mdc`（Always Rule）：每次会话都执行的轻量级读写协议
- 本 Skill：用户主动触发的深度复盘，输出更详细的经验分析
