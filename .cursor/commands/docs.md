# 生成/更新项目文档

根据当前代码生成或更新对应的文档。

## 步骤

### 1. 确定文档范围
判断需要生成/更新的文档类型：
- **API 文档**：接口说明、请求/响应格式、错误码
- **功能文档**：业务流程、功能说明
- **架构文档**：模块关系、数据流、部署架构
- **开发指南**：本地启动、环境配置、编码规范

### 2. 读取现有文档
检查 `docs/` 目录下是否有对应的现有文档，避免重复创建。

### 3. 生成文档内容
- 从代码自动提取接口签名、注释、数据结构
- 补充业务含义（代码注释中没有的）
- 使用 Markdown 格式，结构清晰

### 4. 同步更新
- 如果新增了 API，同步更新 API 文档
- 确保文档与代码保持一致

## 输出格式
- 标题层级清晰（H1 > H2 > H3）
- 代码示例使用代码块
- 接口文档包含：路径、方法、参数、响应示例
