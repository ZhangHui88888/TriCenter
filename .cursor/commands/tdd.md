# TDD 测试驱动开发流程

按照 RED → GREEN → REFACTOR 循环执行：

## 流程

### 1. RED（编写失败的测试）
- 根据需求编写测试用例
- 确保测试运行且**失败**（验证测试有效）
- 测试应该覆盖：正常路径、边界条件、错误处理

### 2. GREEN（最小实现通过测试）
- 编写**最少的代码**使测试通过
- 不要过度设计，只做测试要求的事
- 运行测试，确认全部通过

### 3. REFACTOR（重构优化）
- 在测试通过的保护下优化代码
- 消除重复、改善命名、提取方法
- 每次重构后运行测试确保不破坏功能

## 注意事项
- 每个循环只关注一个小功能点
- 先写接口（测试），再写实现
- Commit 节奏：RED 后不提交，GREEN 后提交，REFACTOR 后提交
