# 基本行为准则

1. 中文回复，保持一致的中文交流
2. 简洁完成，任务完成后删除多余的测试文件，不生成多余文件
3. 智能编辑，优先搜索替换，避免重写整个文件
4. 分段处理，长文件分段创建/编辑，大文件分段读取
5. 先了解再动手，编辑前先获取相关代码上下文
6. 失败不重试，分析失败原因后再行动
7. 文档管理遵循「文档管理规范」，所有文档必须归档到 docs/ 对应子目录并登记到 docs/README.md
8. 功能修改后及时更新 docs/ 中对应文档
9. 禁止执行 git add/commit，由我手动操作
10. 修复控制台警告，运行时不应出现警告
11. 禁止编写未用到的代码，完成后清理无用代码
12. 长耗时命令开新终端窗口，避免阻塞会话
13. 在编写代码时，要避免单个文件过长，合理拆分文件
