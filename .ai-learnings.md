<!-- 经验条目从这里开始 -->

1. **多项目同服务器部署**：TriCenter 和 Booking-miniapp 共用服务器时，必须隔离后端端口（8080/8081）和 Redis database（0/1），否则会冲突。Nginx 用不同子域名路由。

2. **Booking-miniapp 已预留集成配置**：Booking-miniapp 的 `application.yml` 中 `tricenter.base-url` 已配置为 `http://localhost:8081`，两个系统间有 API Key 认证的跨系统调用关系。

3. **SSL 证书共享**：两个项目的 Nginx 配置都引用 `/etc/letsencrypt/live/cz3zx.top/` 下的证书，使用通配符证书可以覆盖所有子域名，无需逐个申请。

4. **Spring Boot 多环境配置**：`application.yml`（公共）+ `application-dev.yml`（开发）+ `application-prod.yml`（生产），通过 `spring.profiles.active` 切换。生产环境敏感信息用 `${ENV_VAR:default}` 注入。

5. **React 组件文件禁止超过 300 行**：EnterpriseDetail 曾膨胀到 6213 行（单体组件 = 9 Tab + 24 Modal + 所有状态/handler），重构花了大量精力。新页面必须从一开始就按 `tabs/` `modals/` `constants.ts` `utils.ts` 拆分，每个 Tab/Modal 独立文件。功能逐步堆积是失控的根源。

6. **AI 编码时主动拆分文件**：AI 倾向于往现有文件追加代码。当文件超过 200 行时，必须停下来评估是否需要拆分，不能等到失控再重构。
